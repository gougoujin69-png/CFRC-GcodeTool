# CFRC G-code Exporter — Excel 插件安装与使用

## 你拿到的文件

| 文件 | 用途 |
|---|---|
| `CFRC_GcodeExporter.bas` | VBA 模块源代码（核心逻辑） |
| `示例数据.xlsx` | 一份带剪丝标记的示例路径，装好就能拿来测 |

> 为什么不直接给 `.xlam`？因为 Excel 的 VBA 工程是二进制（`vbaProject.bin`），不同 Office 版本/语言/位数生成的二进制不兼容，跨机器分发常常一打开就报"工程不可用"。把 `.bas` 在你自己机器上导入并另存为 `.xlam`，**只需要点 5 下，10 秒钟搞定**，而且在你机器上 100% 稳定。下面步骤照做即可。

---

## 第一次安装（仅需做一次，约 2 分钟）

### Step 1. 启用"开发工具"选项卡
打开 Excel → `文件` → `选项` → `自定义功能区` → 右侧勾上 **开发工具** → 确定。

### Step 2. 导入 .bas 模块并另存为加载项

1. 新建一个空白 Excel 工作簿
2. 按 **Alt + F11** 打开 VBA 编辑器
3. 在左侧"工程资源管理器"中右键 `VBAProject (Book1)` → **导入文件…** → 选 `CFRC_GcodeExporter.bas`
4. 关闭 VBA 编辑器，回到 Excel
5. `文件` → `另存为` → 文件类型选 **Excel 加载项 (*.xlam)**
   - 文件名建议：`CFRC_GcodeExporter.xlam`
   - 保存位置 Excel 会自动跳到默认加载项目录（`%AppData%\Microsoft\AddIns\`），**不要改**，直接保存
6. 关闭这个工作簿（不需要保存原 xlsx）

### Step 3. 启用加载项

`开发工具` → `Excel 加载项` → 勾选 **Cfrc_Gcodeexporter** → 确定。

> 现在加载项已经常驻 Excel，每次打开 Excel 都会自动加载。

### Step 4. 把按钮放到快速访问工具栏（推荐，省得每次按 Alt+F8）

1. 右键 Excel 顶部任意位置 → **自定义快速访问工具栏…**
2. 左上"从下列位置选择命令" → 选 **宏**
3. 在列表里找到 `ExportGcodeFromSheet` → 点 **添加 >>** → 右下"修改"可以换图标和显示名（建议改成"导出 G-code"）→ 确定

完成后，Excel 左上角就有一个"导出 G-code"按钮，点一下即可导出。

## 使用方法

### 数据格式

在任意工作表里按 **A=X, B=Y, C=Z, D=Cut** 填数据，第一行可以是表头：

| X (mm) | Y (mm) | Z (mm) | Cut (1=剪丝) |
|---|---|---|---|
| 50  | 50  | 0.2 |   |
| 150 | 50  | 0.2 |   |
| 150 | 150 | 0.2 |   |
| 50  | 150 | 0.2 |   |
| 50  | 50  | 0.2 | **1** |
| 200 | 50  | 0.2 |   |
| …   | …   | …   | … |

- **Cut 列写 1** 表示到达该点后剪丝；其他点留空即可
- 自动识别：`1`、`Y`、`YES`、`TRUE`、`CUT` 都视为剪丝
- Z 突变（`|ΔZ| > 0.05 mm`，可改）会自动判定为"换层"，该段不挤出
- 剪丝点之后到下一个点，自动判定为空驶（不挤出）

### 操作步骤

1. 打开你的数据 xlsx，确保活动工作表就是要导出的那张
2. 点击 **快速访问栏 → 导出 G-code**（或 Alt+F8 → `ExportGcodeFromSheet`）
3. 依次回答 4 个对话框：
   - **(1/3) 工艺参数**：`线宽W, 层高H, 丝径D, 打印速度FP, 空驶速度FT, 剪丝回抽RT, Z阈值`
     默认 `0.4, 0.2, 1.75, 1800, 3000, 0, 0.05`
   - **(2/3) 剪丝宏命令**：默认 `CUT_FIBER`，按你固件实际宏改即可（例如 `M280 P0 S90`）
   - **(3/3) Header**：第一次会问要不要"复用上次保存的"。第一次没有，会弹一个临时表，把你那段 Header 完整粘贴到 **A2 单元格**（已预填默认 Header），改完点确定即可。**以后这段会自动记住**，再导出直接选"是"复用。
   - **是否表头**：如果第 1 行是表头选"是"
4. 选保存路径 → 完成
5. 弹窗会显示：数据点数、剪丝次数、总挤出 E、总路径长度

## 输出 G-code 样例

```gcode
; HEADER_BLOCK_START
; generated by OrcaSlicer 2.2.0 on 2026-04-15 at 15:15:01
; ... (你设的 Header)
G92 E0                      ; 重置挤出机

; ===== Toolpath start (from XLSX) =====
G92 E0                      ; 重置挤出
G1 X50 Y50 Z0.2 F3000 ; 移动到起点
G1 X150 Y50 E1.6611 F1800
G1 X150 Y150 E1.6611 F1800
G1 X50 Y150 E1.6611 F1800
G1 X50 Y50 E1.6611 F1800
; ---- Cut at point 5 ----
CUT_FIBER
G92 E0                      ; 剪丝后重置 E
G1 X200 Y50 Z0.2 F3000 ; travel after cut
...
; ---- Layer 2  Z=0.4 ----
G1 X140 Y100 Z0.4 F3000 ; layer move
...
```

挤出公式（相对模式 M83）：`E = 段长 × W × H / (π·(D/2)²)`

## 修改逻辑 / 二次开发

打开 Excel → Alt+F11 → 在工程资源管理器中找 `CFRC_GcodeExporter.xlam` → `Modules` → `CFRC_GcodeExporter` → 双击编辑。改完保存即可，下次打开 Excel 自动生效。

常见想改的点都集中在文件最上方常量区或 `BuildGcode` / `EmitCut` 两个函数里：

- 改默认参数 → 顶部 `DEF_xxx` 常量
- 改剪丝时序（比如先回抽再走开 1mm 再剪）→ `EmitCut` 子过程
- 加 G-code 尾部命令（关风扇、抬 Z、归零）→ `BuildGcode` 末尾的 `; ===== End` 行之前
- 改坐标偏移 / 缩放 → `BuildGcode` 主循环里 `xc, yc, zc` 赋值后插入
- 加进度提示 → 主循环里加 `Application.StatusBar = ...`

## 故障排查

| 现象 | 原因 / 解决 |
|---|---|
| 按钮点了没反应 | 看是否启用了宏：`文件 → 选项 → 信任中心 → 信任中心设置 → 宏设置 → 启用所有宏` |
| 报"`Cannot find project or library`" | VBA 编辑器 → `工具` → `引用` → 取消所有"MISSING"项 |
| Header 中文变乱码 | 文件已用 UTF-8 无 BOM 写出。如果切片机/打印机要求 GB2312/ANSI，把 `WriteUtf8` 改成 `Charset = "gb2312"` 即可 |
| E 值偏大/偏小 | 检查线宽 W、层高 H、丝径 D 是否正确；公式按面积比，`W=0.4 H=0.2 D=1.75` 时 `e_per_mm ≈ 0.0332` |
| 想看导出前的 G-code 内容 | 在 `BuildGcode` 末尾 `BuildGcode = Join(...)` 之前加一行 `Debug.Print Join(outArr, vbCrLf)`，按 Ctrl+G 看立即窗口 |
